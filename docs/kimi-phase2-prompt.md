## LOCKED decisions (ratified — do NOT re-open; bake these in)

- **BORROWED handle model.** `NodeRef<'tree>` borrows the `DataTree` **immutably**; the `*lyd_node` **never** crosses the public boundary (hexagonal). Iterators (`children`/`siblings`/`traverse`/`select`) materialize an **ordered `Vec` of owned domain handles in ONE coarse FFI walk** — **no per-node FFI** inside the loop. Reads borrow `&`; **ALL** mutation is path-addressed on `&mut DataTree` (`new_path`/`set_value`/`remove_path`). Because mutators take `&mut` and reads borrow `&`, the borrow checker forbids read-while-mutate — **no stale handle is possible in safe Rust.** The `{gen}` generation counter and the runtime `E0007 Stale` check are a **Go-mirror concern only** — do **not** add a gen counter or any stored-pointer escape hatch in Rust.
- **Native runtime model.** `validate(&mut self)` is a runtime call — **no** `Validated`/`Unvalidated` typestate. `ContextBuilder` → frozen `Context` typestate already shipped; do not re-derive it.
- **Ordering stays exclusively on `UserOrdered`/`UserOrderedLeafList` (I1).** The CRUD layer **never** reorders. `new_path` must insert at the **schema-correct position** (libyang does this natively) so a tree built out of source order already serializes in declaration order (**I2**).
- **`Value` aligns with Phase-1 `BaseType`.** One `Value` variant per `BaseType` (`schema.rs`). `Value` (data) and `BaseType`/`ResolvedType` (schema) stay **separate** — do not conflate. `decimal64` has no std type: model it as a newtype over `{ raw: i64, fraction_digits: u8 }` — **not** `rust_decimal`.

## CRITICAL FFI gotchas — the Phase-2 analogue of the 1b pointer-array trap

These are the load-bearing traps from the verified catalog. Inline the safe pattern; do **not** rediscover them. **Verify every struct field name / type / pointer-depth in `bindings.rs` and the vendored header BEFORE the unsafe read** (the 1b win came from distinguishing array-of-structs from array-of-**pointers**).

1. **`lyd_get_value` and `lyd_child` are `static inline` — ABSENT from `bindings.rs`.** Calling `lyd_get_value` will **fail to link**. `spec/api.md`/docs say `value_str()` is "(AGENTS-mandated `lyd_get_value`)" — that means *reimplement its body in Rust*, not bind the symbol. Safe path: for a term node read `(*term).value._canonical` (`*const c_char` at offset 0 of `lyd_value`); if it is **NULL** (it is a lazy cache) call the **real** binding `lyd_value_get_canonical(ctx, &(*term).value)` (`bindings.rs:5355`). The existing `lyd_child` reimpl (`adapter.rs:1355`) casts to `lyd_node_inner` **unconditionally** — that is **UB on a leaf** (offset 56 is `lyd_value`, not a child pointer). **Fix the existing helper in place** by gating it (so the trap can't be reintroduced): gate every `.child` read on `(*node).schema` non-null **AND** `nodetype & (LYS_CONTAINER|LYS_LIST|LYS_RPC|LYS_ACTION|LYS_NOTIF)`; gate every `.value` read on `nodetype & LYD_NODE_TERM (== LYS_LEAF|LYS_LEAFLIST)`. Check `(*node).schema->nodetype` **before** any subtype cast.

2. **`lyd_node` sibling chain is circular on `.prev`.** `.next` (offset 24) is NULL-terminated; **`.prev` is NEVER null** — the first node's `.prev` points to the **last** sibling. **Always iterate forward via `.next`**; a `while !prev.is_null()` loop never terminates. `.next`/`.prev`/`.parent`/`.schema` are single `*mut`/`*const` (no double-deref). Children live on the **inner** subtype (`lyd_node_inner.child` @56); values on the **term** subtype (`lyd_node_term.value` @56). The same circular-`.prev` rule applies to the `ly_err_item` chain.

3. **`lyd_find_xpath` ALLOCATES an `ly_set` the caller MUST free — with a NULL destructor.** `lyd_find_xpath(ctx_node, xpath, &mut *mut ly_set)`. `ly_set { size:u32@0, count:u32@4, union { dnodes: *mut *mut lyd_node, ... } @8 }`. The matched nodes are an **array of POINTERS** (`*mut *mut lyd_node`) — index `(*set).__bindgen_anon_1.dnodes.add(i)` then deref. **Copy out all `count` pointers into owned domain snapshots in ONE walk, THEN `ly_set_free(set, None)`** — pass `None` (NULL destructor). Passing **any** destructor frees the matched nodes, which still belong to the live `DataTree` → use-after-free + double-free on Drop. The set is **not** null-terminated (use `.count`, not a sentinel) and may be allocated-but-empty (`count==0`) — still must be freed. An XPath that returns a number/string/boolean (not a node-set) still allocates an `ly_set` with `count==0` — `select()` over it yields an **empty `NodeSet`**, not `E0006`.

4. **`lyd_change_term` returns THREE codes, not two — and `LY_EEXIST` must be resolved BEFORE coding.** `LY_SUCCESS` = value changed; `LY_ENOT` = identical no-op; **`LY_EEXIST` = same value but the default-flag was cleared (a real state change)**; any other code (notably `LY_EVALID`) = real error. Map `LY_SUCCESS → Ok(true)`, `LY_ENOT → Ok(false)`, else `Err(E0006)`. **Decide `LY_EEXIST` up front** by reading `docs/sdk-api-design.md` §C + `tree_data.h` for `lyd_change_term`: pick **one** mapping (`Ok(true)` is the lean — a default→explicit transition *is* a change), encode it as a named constant/comment, and assert that exact branch in `set_value_reports_changed`. Do **not** leave it a runtime coin-flip. A naive `if rc != LY_SUCCESS { Err }` is **wrong**. `term` must be a leaf/leaf-list — gate on `LYD_NODE_TERM` before the cast.

5. **`lyd_new_path`'s out-param `node` is the FIRST created node, NOT the addressed leaf.** For `/a/b/c` it points at `a`, not `c`. To return a `NodeAddr` that addresses the leaf you set, either use **`lyd_new_path2`** (separate `new_parent` **and** `new_node` out-params; `new_node` = last created / the list instance) **or** re-resolve via `lyd_find_path(path)` after creation. **On an empty tree** (`new_data`) `parent` is NULL and **`ctx` is required** — `RawDataTree` only holds `*mut lyd_node` and **cannot reach the ctx** (`RawContext.ptr` is private with no accessor). See **Slice 0**: the ctx pointer must be plumbed into `RawDataTree` first; `new_path`, the `value_str` canonical fallback, the validate error-walk, and `add_defaults` all depend on it. When `self.ptr` was null, **write the returned root back into `RawDataTree.ptr`** or the subtree leaks and the tree stays empty. An absolute path may insert a new top-level sibling **before** the old first sibling — **re-anchor via `lyd_first_sibling`** after `new_path` on a non-empty tree; never cache the old first-sibling pointer.

6. **`remove_path` = unlink-then-free; pick the RIGHT free function.** `lyd_unlink_tree(node)` **detaches but does NOT free** — the subtree becomes caller-owned (drop it = leak). `remove_path` = `find_path` → `lyd_free_tree(node)` (`lyd_free_tree` unlinks **and** frees just that subtree; the parent tree stays valid). `unlink_path` = `find_path` → `lyd_unlink_tree`, then wrap the detached `*mut lyd_node` in a **fresh `RawDataTree`** so exactly one Drop owns it (mirror the existing `into_raw` pattern that nulls `self.ptr`). **NEVER** call `lyd_free_all` on a still-linked interior node — it frees parents **and** all siblings → dangling `DataTree` root → double-free on Drop. `DataTree`'s own Drop calls `lyd_free_all` on the canonical root **once**. **Same-context invariant:** a `DataTree` returned by `unlink_path` may only be re-inserted (via `UserOrdered::insert_*`) into a tree from the **same `Context`** — cross-context re-insertion is undefined behavior in libyang. Do not write a test that mixes contexts; document this constraint on `unlink_path`.
