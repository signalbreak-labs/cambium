# Overnight goal — Phase 1: rich schema introspection + TypeInfo (Rust)

> Unattended autonomous run, started 2026-06-13 evening. Branch: `feat/cambium-sdk-phase1`.
> Contract for the run. Authoritative scope: `spec/api.md` (the ratified contract) +
> `docs/sdk-api-design.md` §B (full signatures). Decisions are locked — do not re-open them.

## Goal
Close the single biggest goyang-parity gap: replace the coarse `SchemaTree`/`SchemaNode`/`LeafType`
with goyang-grade schema introspection — borrowed-handle metadata + a real `TypeInfo`/`ResolvedType`
sum type — implemented Rust-first via TDD. This unblocks real typed codegen (enums, identityref,
leafref, union) and the data→schema bridge.

## Hard gates (NON-NEGOTIABLE — a slice that can't satisfy these is reverted and deferred, never committed)
1. **Green bar before every commit:** `cargo build --workspace`, `cargo clippy --workspace --all-targets -- -D warnings`, `cargo test --workspace`, `cargo run -p conformance-runner`, and `(cd go && go build ./... && go vet ./... && go test ./...)` ALL pass. If a slice cannot reach green, `git checkout -- <its files>` and record it in the handoff as deferred — do NOT commit red, do NOT hack around it, do NOT weaken/`#[ignore]` an existing test to go green.
2. **Do NOT touch the serialization / ordering output path.** Phase 1 is read-side schema introspection only. The existing `/conformance` goldens MUST stay byte-identical — that is the proof ordering was not harmed. Never regenerate, edit, or blind-accept a golden file.
3. **TDD, red first:** write the failing test (named per `spec/api.md` §Acceptance tests) and watch it fail before production code. New goldens/oracle values are generated by libyang/yanglint/pyang and reviewed in-diff — never hand-authored.
4. **Keep the existing public API working.** Keep `SchemaTree`/`SchemaNode`/`LeafType`/`Context::schema_tree`/`Context::new` as a deprecated-but-present thin view so M0–M3 tests and the v1 codegen keep compiling. Additive/parallel, not a hard break this run.
5. **Hexagonal:** no libyang/cgo types in `cambium-core`'s public API. New raw extraction lives in `cambium-libyang-sys`; map to domain types at the boundary in one coarse walk (no per-node FFI).
6. **Rust house style:** no `unwrap`/`expect` outside tests; `thiserror`/`Result`; enums over bool+Option; `#![deny(missing_docs)]`. `cargo fmt --all` before the final commit.
7. **Go is untouched this run** (Go parity is Phase 4). It must still build+test green (gate 1) but gets no new features.
8. **Flag every FFI/`unsafe` change** in the handoff under a "NEEDS HUMAN REVIEW" heading (the C-boundary human-verification rule).

## Work plan (ordered; commit each green slice separately on the branch)
- **Slice A — adapter: low-risk node metadata.** Extend `RawSchemaNode` + the adapter walk to capture `lysc_node` flags: config (R/W vs R/O), status, mandatory, presence-container, description, units, default, min/max-elements; and the precise leaf **base type** (the 19 RFC-7950 builtins) + typedef name. TDD at `cambium-libyang-sys/tests` and `cambium-core`.
- **Slice B — adapter: rich type constraints (HIGHER FFI RISK — proceed only if confidently green).** Extract `lysc_type` substructs: integer/decimal64 ranges + fraction-digits, string length + patterns (+ error-app-tag), enumeration `(name,value)` pairs in declaration order, bits `(name,position)`, identityref bases, leafref target path, union member types (recursive). If any sub-part can't be verified green, defer it with notes — do not guess at C struct layouts.
- **Slice C — core domain types.** `Status`/`Config`/`OrderedBy`/`BaseType` enums; `TypeInfo` → `ResolvedType` sum type (illegal constraint combos unrepresentable); `Identity` with transitive `derived()` closure; enrich `SchemaNode` getters (description/config/status/mandatory/presence/units/default/min-max/list_keys-as-nodes). Map from Slice A/B raw data at the boundary.
- **Slice D — context typestate + module handles.** `ContextBuilder` (mutable) → `build()` → frozen `Context` (`Send + Sync`); `Module` handle (name/namespace/prefix/revision/is_implemented/feature_value); module enumeration. Keep the old `Context::new()`/`schema_tree()` deprecated-present.
- **Slice E — facade + prelude.** Re-export the new public types via `cambium` and a `cambium::prelude` (today the facade omits `SchemaTree`/`DataTree`/`UserOrderedList`/`RuleCode` — fix that).

Reach as far down this list as the green bar allows. Partial, green, well-documented progress beats a broad red sweep.
